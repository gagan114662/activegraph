## Justification

### Staleness failure mode (verbatim answer to the yaml's ask)

Each entry carries an `expiry_commit_ref` (frame id or commit hash that,
once landed, MUST trigger removal) and an `expiry_date` (hard date
backstop). The gate enforces both:

1. **Hard expiry:** if `expiry_date < today`, the gate fails closed with
   `AllowlistEntryExpiredError(flag=<name>, expired_on=<date>)`. The
   entry cannot be silently extended ŌĆö extension requires a commit that
   bumps the date, which surfaces in the next PR review.

2. **Resolution expiry:** if `expiry_commit_ref` is the id of a closed
   frame (e.g., `t3-implement-cli-set-flag`), the gate fails closed with
   `AllowlistEntryResolvedError(flag=<name>, resolving_frame=<id>)`.
   This catches the "T3 shipped, we forgot to remove `--set` from the
   allowlist" case.

3. **Forgery resistance:** entries missing either `rationale` or
   `expiry_date` cause the gate to fail closed at load time
   (`AllowlistEntryMalformedError(flag=<name>)`). The schema cannot be
   silently softened by a careless contributor.

Staleness pressure is therefore mechanical, not social. The gate
re-fails every PR after expiry, forcing either resolution or explicit
extension ŌĆö no "we'll get to it" decay path.

### Visibility guarantee for pending claims (verbatim answer to the yaml's ask)

Each pending claim is at one path: `activegraph/cli_flag_drift_allowlist.toml`.
This guarantees:

- **Diff-reviewable:** every PR that adds, modifies, or removes a
  suppression touches this file. Code review sees the full set.
- **Greppable:** `grep "^\[\[" cli_flag_drift_allowlist.toml | wc -l`
  gives the suppression count. CI can publish this number.
- **Auditor-readable:** a single file beats 8+ markdown files scattered
  across `docs/`. An auditor asking "what drift are we currently
  tolerating?" reads one file, not the whole doc tree.
- **Tooling-amenable:** a `scripts/list_drift_allowlist.py --expiring-soon`
  helper is one TOML parse away. Inline HTML comments would require
  cross-document scanning to answer the same question.

## Schema (informational, not binding on Code Owner)


## Rejected: (a) inline HTML comments

Three failure modes that disqualify this option:

1. **Decay-by-relocation:** when a doc is restructured, the comment
   travels with the doc text ŌĆö but if the doc is split across files
   or the example moves, the suppression silently follows unrelated
   content. Inline suppressions accumulate without review pressure.

2. **No expiry mechanism:** HTML comments are static strings. Adding
   an `expires=` attribute is possible but requires custom parsing on
   top of standard markdown ŌĆö at which point we've reinvented (b)
   without the central-file benefit.

3. **Auditor blindness:** "what drift are we tolerating right now?"
   requires scanning every `.md` file in the corpus. Cannot be
   answered from a single file diff.

The locality benefit (suppression travels with the doc) is real but
small ŌĆö every PR that touches a suppression already touches the file
that contains it, so review proximity is already guaranteed at the
PR-diff level regardless of whether the suppression lives in the doc
or in a TOML.

## Rejected: (c) refuse to allowlist, ship T3 first

Inverts the gauntlet's intended sequence. Per the frame's goal at line
22ŌĆō25, t2 must "pass against current main with an explicit allowlist
for T3-pending flags" ŌĆö main today has `--set` / `--memo` / `--search`
in docs but not in code. Refusing the allowlist forces one of:

- **t2 cannot ship until t3 ships:** sequencing inversion. t2's purpose
  is to *catch* this drift class going forward; deferring it until t3
  resolves the current instance leaves no gate in place when the next
  instance happens.
- **Delete the doc references first:** scrubs evidence of the
  intended user-facing surface from the docs, harming roadmap signal
  for users reading current docs.

Neither is acceptable. The allowlist mechanism is necessary; the only
question is which form, and (b) wins on staleness pressure and
auditor visibility.
