 What just happened
----------------------------------------------------------------------------

  1. You loaded a pack. A pack is a Python package that registers
     object types, behaviors, tools, and prompts. The Diligence pack
     ships with activegraph.

  2. The runtime received three goals (one per company) and reactive
     behaviors fired automatically as objects appeared on the graph.
     You did not write a workflow. The behaviors are pattern-matched
     against event types and object shapes.

  3. Every LLM call was served by the bundled fixture provider
     (RecordedDiligenceProvider), so the run is deterministic and runs
     offline. The trace above shows cost and latency on each llm.responded
     line — those are the fixture's recorded numbers. Production runs
     against a real provider would show real costs and latencies.

     (The framework also has a separate replay cache that records
     llm.responded events and serves them back under strict-replay mode
     or in-process Runtime.fork() — that's where `cache_hit=true` appears
     in the trace. Different layer from the provider; see the
     concepts/replay and concepts/forking pages for the deep dive.)

  4. Each memo cites evidence for every claim, surfaces at least one
     risk, and either lists open contradictions or states explicitly
     that none were found. This is the pack's "verifiable memo bar" —
     a memo without these properties is a bug in the pack.

  5. Every event is on disk. The full causal chain from goal to memo
     is reconstructable from the event log alone.
