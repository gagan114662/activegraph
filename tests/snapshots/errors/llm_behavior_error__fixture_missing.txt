LLMBehaviorError: llm.fixture_missing: no recorded fixture for prompt_hash=a1b2c3d4 in /tmp/fixtures

What failed:
  The RecordedLLMProvider has no fixture for this prompt:
    no recorded fixture for prompt_hash=a1b2c3d4 in /tmp/fixtures

Why:
  RecordedLLMProvider replays a directory of recorded LLM responses keyed by prompt content hash. A missing fixture means the live prompt's hash doesn't match any recorded response — either the prompt changed since the fixtures were recorded (a behavior edit, a template change, a tool input difference), or this is a new prompt that was never recorded.

How to fix:
  Re-record the fixture from a live run with the current prompt:
      1. Switch to AnthropicProvider (set ANTHROPIC_API_KEY)
      2. Run the goal once to produce live LLM responses
      3. The provider records each response to the fixture directory
      4. Subsequent runs against RecordedLLMProvider replay them

  Or diff the prompt against the recorded version to find the drift.
