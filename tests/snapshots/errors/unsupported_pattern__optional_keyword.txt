UnsupportedPatternError: the OPTIONAL keyword is not supported in the v0.7 Cypher subset

What failed:
  The pattern uses the OPTIONAL keyword. The v0.7 subset refuses this feature at registration time — long before any match runs.

Why:
  The pattern subset is deliberately small and exhaustively testable. A fuzzy superset of Cypher would let patterns appear to match input they did not actually match, which would break the audit trail that pattern-driven behaviors preserve. See CONTRACT v0.7 #8 for the locked subset.

How to fix:
  OPTIONAL MATCH expresses 'match if present, else null.' The runtime
  does not have a null binding. Register a second behavior whose
  pattern is the optional sub-pattern.
