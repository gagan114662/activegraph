ApprovalNotFoundError: no pending approval named 'approval_999'

What failed:
  runtime.approve('approval_999') could not find a pending approval with that id.
    There are currently 2 pending approvals; none of them match 'approval_999'.

Why:
  Approval ids are generated by the runtime when a behavior calls ctx.propose_object under a gating policy (memo_approval, risk_approval, etc.). Each id is unique to its runtime instance and is consumed when approve() succeeds. A miss means either the id is stale (already approved or denied), the runtime instance is fresh (approvals don't carry across Runtime.load), or the id is a typo. The framework refuses to no-op an unknown id because that would silently corrupt the approval audit trail.

How to fix:
  List the currently-pending approvals:
      rt.pending_approvals()
  to confirm what's actually pending. The diligence demo's
  step_2_approval_demo (examples/diligence_real_run.py)
  shows the canonical pattern: enumerate, then approve by id.
